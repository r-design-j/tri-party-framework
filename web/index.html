不用自己装

这个页面不是让用户学习一堆命令，而是给你一段可以直接丢给 AI agent 的安装委托。

普通用户怎么理解

triparty 是一套本机协作框架。你把安装任务交给 agent，它负责拉取项目、安装规则、检查环境，再把结果告诉你。

你只要复制

点击“复制给 Agent”，把那段话发给能操作本机终端的 AI agent。

Agent 去安装

agent 会 clone GitHub 项目、补权限、安装 bootstrap，并运行检查。

结果会回报

安装后它会告诉你路径、检查结果，以及缺少 Claude、Gemini 或认证时该补什么。

网页不越权

网页不会偷偷写你的电脑。真正安装必须由你授权的本机 agent 或终端完成。

复制给 Agent

最推荐的用法：复制一段委托话术给你的 AI agent，让它在本机安装并回报结果。

拉取项目

Agent 会从 GitHub 拉取 triparty，并在本机建立一个可重复使用的框架目录。

你不需要手动下载 ZIP，也不需要逐条复制命令。
会发生 02

写入规则

Agent 会安装全局发现规则，让之后的新会话也能找到 triparty，不需要每次重新解释。

Claude Code 会有 `/triparty` 和 `/tp`，Codex 可用 `triparty` 命令。
会发生 03

回报结果

Agent 会运行安装检查，并告诉你是否可用；如果缺少 Claude、Gemini 或登录认证，它会明确报出来。

重点是知道“装好了没有”，而不是让用户自己猜。

案例 01 · 已实现

运行状态检查

把一次协作运行的结果变成可读、可检查、可解释的页面。

角色状态解释器
数据源运行状态文件
平台静态 Web
状态已可体验

真实问题

用户或 agent 看到一堆运行文件时，容易把“部分完成”误读成“已经完成”。

怎么修

只读取核心状态字段。页面负责解释，不自己发明成功标准。

发生了什么

点击“加载完成样例”或“加载未完成样例”，页面会改写状态卡片和底部提示。

结果

它不是纯说明文字，而是能解析运行状态的前端检查器。

约束

它不会直接读取本机文件，也不会直接调用模型；这样避免把浏览器页面变成未经审计的执行入口。

下一步

冻结运行状态格式后，可以把粘贴输入升级为本地只读加载，仍然不直接执行 agent。

案例 02 · 已实现

发布前检查

把发布前判断变成可解释界面：为什么能发、为什么不能发、缺哪一方证据。

角色发布前检查
数据源发布检查 + 状态文件
平台CLI 生成 / Web 展示
状态已可体验

真实问题

“能不能发布”不能靠聊天结论，必须依赖文件证据、hash 和完成标记。

怎么修

页面把发布前命令、状态摘要和“未完成协作”的阻断语放在同一视图里。

发生了什么

完成样例会显示可支持发布声明；未完成样例会明确写出不能升级为完整协作。

结果

产品化价值在于降低判断成本，而不是绕过真正的发布前检查。

约束

页面不会把单方输出包装成三方完成。缺证据时必须阻断。

下一步

补全 artifact SHA256、sanitizer_version、retry/timeout 的可视字段，让门禁解释更完整。

案例 03 · 已实现

失败恢复

把最常见的 triparty 失败变成中文恢复路线，而不是让用户在 stderr 和日志里猜。

角色排障向导
数据源错误码映射
平台Web + CLI
状态已可体验

真实问题

登录认证、运行噪声、文件不一致、未完成协作是不同问题，混在一起会导致错误修复路径。

怎么修

排查工具提供错误码下拉选择，输出对应中文解释和下一条命令。

发生了什么

选择错误码后，页面会立即给出认证检查、环境检查、发布前检查或重跑建议。

结果

这是可体验功能，不是纯文档。它让新用户知道下一步该回到哪个真实执行面。

约束

恢复建议不会自动执行，避免误触发模型、网络或文件写入。

下一步

把映射表从内联 JS 抽到稳定 JSON，并由 regression 覆盖错误码文案。

案例 04 · 待接入

开发接入

真正的产品化不是让网页直接变 agent，而是让 UI 清楚展示本地执行层的边界。

角色接入说明
数据源本地服务
平台本地服务
状态下一阶段

真实问题

如果网页看起来像能一键调用三模型，用户会误以为浏览器就是执行层，反而破坏可信边界。

怎么修

UI 只展示约定、复制请求、解释返回。真实调用仍在本机执行层。

发生了什么

当前页面已经把接入清单写清楚：冻结状态字段、补样例、只读接入本地服务。

结果

它能承接产品化演示和外部接入沟通，但不越权承诺浏览器端能完成三方运行。

约束

本地服务默认只在本机运行，不能被 UI 文案包装成公网 SaaS。

下一步

补 `ui-scope.md`、`surface-binding.yaml` 或等价文档，再接入只读 API。

排查工具

先运行 `scripts/triparty-gemini-auth-doctor.sh`。如果仍是 interactive-auth-required，先在 Gemini CLI 完成登录，再重新跑 `scripts/triparty.sh preflight`。

  • 把当前静态页面接入本地只读服务，不直接调用模型。
