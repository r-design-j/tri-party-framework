产品定位

真实执行仍在 Codex、Claude Code、Gemini CLI、HTTP/MCP adapter 中。网页只负责可视化、解释和引导。

它到底是什么

triparty UI 是可视化使用说明手册、状态检查器、操作向导和失败恢复导航。它帮助人理解协议，不冒充协议。

是仪表盘

读取 `state.json`、展示 phase、true_triparty_ready、每个模型的 review / cross-audit 状态。

是说明手册

把 triparty 的角色分工、发布门禁、partial 规则，用中文讲清楚。

是操作向导

根据任务生成 CLI 命令，帮助用户回到真实 agent / terminal 中执行。

不是 agent

不直接调用 Codex、Claude、Gemini，不绕过 CLI、Python、HTTP/MCP adapter。

案例详情

案例 01 · 已实现

Run Inspector

面向第一次使用 triparty 的人，把一次 run 的结果从命令行文件变成可读、可检查、可解释的页面。

角色状态解释器
数据源state.json
平台静态 Web
状态已可体验

真实问题

用户看见一堆 run artifact 时，不知道哪些字段代表“真三方”，哪些只是 probe 成功，容易把 partial run 误读为完整结论。

怎么修

只读取 `true_triparty_ready`、party review、cross-audit、Gemini auth、errors 等核心字段。UI 不新增协议规则。

发生了什么

点击“加载真三方”或“加载部分结果”，页面会改写状态卡片和底部提示，让用户马上看到门禁差异。

结果

它已经不只是说明文字，而是能解析 JSON 的前端检查器。真实运行时只需要把 run 目录里的 `state.json` 粘贴进来。

约束

它不会直接读取本机文件，也不会直接调用模型；这样避免把浏览器页面变成未经审计的执行入口。

下一步

冻结 run manifest schema 后，可以把粘贴输入升级为 HTTP adapter 只读加载，仍然不直接执行 agent。

案例 02 · 已实现

发布门禁

把 release-gate 的判断变成可解释界面：为什么能发、为什么不能发、缺哪一方证据。

角色发布前检查
数据源release-gate + state.json
平台CLI 生成 / Web 展示
状态已可体验

真实问题

“能不能发布”不能靠聊天结论，必须依赖文件证据、hash、来源标签和 completion marker。

怎么修

页面把发布前命令、状态摘要和 partial 阻断语放在同一视图里，用户可以复制命令回 terminal 执行。

发生了什么

真三方样例会显示可支持发布声明；partial 样例会明确写出不能升级为真三方。

结果

产品化价值在于降低判断成本，而不是绕过 release-gate。真正的 gate 仍由脚本执行。

约束

页面不会把 Codex 行渲染成“已完成 review”。Codex 在三方结论中负责当前会话 synthesis。

下一步

补全 artifact SHA256、sanitizer_version、retry/timeout 的可视字段，让门禁解释更完整。

案例 03 · 已实现

失败恢复

把最常见的 triparty 失败变成中文恢复路线，而不是让用户在 stderr 和日志里猜。

角色排障向导
数据源错误码映射
平台Web + CLI
状态已可体验

真实问题

Gemini auth、runtime noise、hash mismatch、partial run 是不同问题，混在一起会导致错误修复路径。

怎么修

Playground 提供错误码下拉选择，输出对应中文解释和下一条命令，避免把 partial 说成 true。

发生了什么

选择错误码后，页面会立即给出 doctor、preflight、release-gate 或重跑 review 的建议。

结果

这是可体验功能，不是纯文档。它让新用户知道下一步该回到哪个真实执行面。

约束

恢复建议不会自动执行，避免误触发模型、网络或文件写入。

下一步

把映射表从内联 JS 抽到稳定 JSON，并由 regression 覆盖错误码文案。

案例 04 · 待接入

Adapter 接入

真正的产品化不是让网页直接变 agent，而是让 UI 清楚展示 CLI、Python、HTTP/MCP adapter 的边界。

角色接入说明
数据源HTTP/MCP adapter
平台本地服务
状态下一阶段

真实问题

如果网页看起来像能一键调用三模型，用户会误以为浏览器就是执行层，反而破坏可信边界。

怎么修

UI 只展示 contract、复制请求、解释返回。真实调用仍在 CLI、Python、HTTP/MCP adapter。

发生了什么

当前页面已经把接入清单写清楚：冻结状态字段、补 fixtures、只读接入 adapter。

结果

它能承接产品化演示和外部接入沟通，但不越权承诺浏览器端能完成三方运行。

约束

HTTP adapter 默认 loopback，本地服务也不能被 UI 文案包装成公网 SaaS。

下一步

补 `ui-scope.md`、`surface-binding.yaml` 或等价文档，再接入只读 API。
